# Security Policy

## Reporting a Vulnerability

If you discover a security issue in PingFlow, please report it privately.

Email: `security@pingflow.example.com`

When submitting a report, include:

- A description of the issue
- Steps to reproduce it
- The affected version or commit
- Any proof-of-concept or exploit details

## Supported Versions

The latest `main` branch is the actively supported version.

## Disclosure Policy

Please do not disclose security issues publicly until they have been addressed.

## Maintenance

Security fixes will be prioritized and released as soon as is practical.

## Non-security issues

For bugs, feature requests, and general support, please use the issue tracker instead of this security channel.
